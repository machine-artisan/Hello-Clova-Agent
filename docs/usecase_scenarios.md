# Use Case Scenarios — 한국어 키워드 기반 덱 생성

## 개요

사용자가 **업무 도메인 키워드를 선택**하면, LangGraph 에이전트가
해당 맥락에 최적화된 Reveal.js 발표 슬라이드를 자동 생성합니다.

---

## 핵심 사용자 질문

> "내가 오늘 보고해야 하는 주제로 발표 자료를 5분 만에 만들 수 있을까?"

---

## Persona별 시나리오

### Persona A — 공공기관 담당자

| 항목 | 내용 |
|------|------|
| 역할 | 중앙부처 정책 담당 주무관 |
| 상황 | 내일 오전 국장 보고, 자료 작성 시간 2시간 |
| 목표 | 정책 현황 + 추진 계획 슬라이드 10장 |
| 제약 | 국산 LLM 사용 의무 (HCX SEED) |

**키워드 선택 흐름 (Gradio UI 기준):**
```
[도메인 선택]     공공행정
[세부 키워드]     디지털전환, 전자정부, 민원처리 자동화
[슬라이드 수]     10장
[보고 대상]      국장 (비전문가 청중)
[강조 사항]      예산 효율화, 추진 일정
        ↓
[에이전트 실행]
        ↓
목차 생성 → 내용 작성 → HTML 출력
```

**생성 결과 예시:**
- 슬라이드 1: 디지털전환 추진 배경 및 필요성
- 슬라이드 2: 현황 진단 (AS-IS)
- 슬라이드 3-6: 3대 핵심 과제
- 슬라이드 7-8: 추진 일정 및 예산
- 슬라이드 9: 기대 효과
- 슬라이드 10: 맺음말

## Phase 2 구현을 위한 변경 포인트

### `agent/nodes/input_parser.py`
현재: 자유 텍스트 파싱
변경: 구조화된 JSON 입력 수용
```python
# 현재
def parse_input(state: DeckState) -> DeckState:
    user_prompt = state["user_prompt"]  # 자유 텍스트

# 변경
def parse_input(state: DeckState) -> DeckState:
    req = state["structured_input"]  # 키워드 + 도메인 + 대상
    # domain, keywords, audience, num_slides 추출
```

### `agent/prompts.py`
도메인별 시스템 프롬프트 분기:
```python
DOMAIN_PROMPTS = {
    "공공행정": "공문서 양식에 맞게, 격식체로, 정책 용어를 사용하여...",
    "스타트업":  "데이터 중심으로, 임팩트 있게, 숫자로 말하는 방식으로...",
    "학술/연구": "논리적 흐름으로, 기술 용어를 정확히 사용하여...",
}
```

### `ui/app.py`
Gradio Blocks로 키워드 선택 UI 추가:
```python
with gr.Blocks() as demo:
    domain = gr.Radio(["공공행정", "스타트업", "학술/연구", "금융"], label="도메인")
    keywords = gr.CheckboxGroup([...], label="키워드")
    audience = gr.Radio(["임원", "동료", "투자자"], label="청중")
    num_slides = gr.Slider(5, 20, value=10, label="슬라이드 수")
    btn = gr.Button("덱 생성")
```

## 우선순위 Use Case

구현 우선순위 기준: 사용 빈도 × 시간 절감 효과

| 순위 | Use Case | 도메인 | 슬라이드 수 |
|------|----------|--------|-------------|

| 5 | 투자 피치덱 | 스타트업 | 7-10장 |
